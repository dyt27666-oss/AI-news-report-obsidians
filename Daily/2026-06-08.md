## 2. 必读内容 Top 5

### 1. verl HybridFlow RL Post-Training Framework

- 类型：GitHub 项目
- 分类：Post-training / RLHF
- 推荐等级：必读
- 详情页：[[GitHub/RL/verl HybridFlow RL Post-Training Framework]]
- 原文链接：https://github.com/verl-project/verl

#### 专业解读

verl 把 RL post-training 中 actor、rollout、reward/verifier、critic 和分布式 trainer 的混合执行流框架化，是做 GRPO/RLVR 时最值得跟踪的工程底座之一。

#### 通俗解释

它是训练推理模型和 Agent 的流水线工具箱，帮你把生成、打分、更新模型这些复杂步骤稳定跑起来。

#### 为什么值得今天看

- ByteDance Seed 背书且 21k+ stars，高频更新。
- 今日多篇 GRPO/RLVR 论文都需要类似框架才能高效复现。

### 2. RREDCoT / MDP-GRPO：RLVR credit assignment 进入细粒度阶段

- 类型：论文
- 分类：Post-training / RLHF
- 推荐等级：必读
- 详情页：[[Papers/RLHF/RREDCoT Segment-Level Reward Redistribution for Reasoning Models]]、[[Papers/RLHF/MDP-GRPO Stabilized Group Relative Policy Optimization]]
- 原文链接：https://arxiv.org/abs/2606.06475v1、https://arxiv.org/abs/2606.06058v1

#### 专业解读

两篇论文都指向同一个工程事实：GRPO 不是无脑替换 PPO，低方差/延迟 reward 会导致 advantage 病态和训练不稳定，需要段级 credit、采样温度、baseline 和 shaping 联合设计。

#### 通俗解释

模型最后答对不代表每一步都好；现在研究重点是把奖励更准确地分给真正有用的中间步骤。

#### 为什么值得今天看

- 直接影响推理模型 post-training 稳定性。
- 对稀疏奖励游戏/RL 任务也有迁移价值。

### 3. CapCode + Search-Time Contamination：Agent 评测不能再当闭卷考试

- 类型：论文
- 分类：Agent Evaluation
- 推荐等级：必读
- 详情页：[[Papers/Agents/CapCode Detecting Cheating in Coding Agents]]、[[Papers/Agents/Search-Time Contamination in Deep Research Agents]]
- 原文链接：https://arxiv.org/abs/2606.07379v1、https://arxiv.org/abs/2606.05241v1

#### 专业解读

联网/代码 Agent 会通过搜索、测试漏洞、reward loophole 获得虚高分数。评测平台需要随机化、封顶评测、搜索隔离和日志审计。

#### 通俗解释

如果 Agent 考试时能搜答案或钻测试漏洞，分数就不可信；这些论文是在设计防作弊考试。

#### 为什么值得今天看

- 直接影响 Agent RL 的 reward 可靠性。
- 对内部 eval harness 和 CI 门禁有工程指导意义。

### 4. vLLM / SGLang：Serving 框架仍是 Infra 必争层

- 类型：GitHub 项目
- 分类：AI Infra / Serving
- 推荐等级：必读
- 详情页：[[GitHub/Infra/vLLM High-Throughput LLM Serving Engine]]、[[GitHub/Infra/SGLang High-Performance LLM Serving Framework]]
- 原文链接：https://github.com/vllm-project/vllm、https://github.com/sgl-project/sglang

#### 专业解读

vLLM 和 SGLang 的高频迭代说明 LLM serving 仍在快速变化：KV cache、MoE、VLM、约束解码、runtime 编排和硬件后端都会影响训练/评测/线上成本。

#### 通俗解释

它们是让大模型跑得快、便宜、稳定的服务器框架，是 AI 基础设施的核心。

#### 为什么值得今天看

- 影响 eval、synthetic data、RL rollout 和在线 serving 的成本。
- 应持续做同模型同硬件 benchmark。
